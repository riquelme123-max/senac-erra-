# minicenario 
um dono de uma loja de ferramentas está com um grande problema com a distribuição dos itens em seu estoque. ele precisa de algum programa para poder organizar os seus produtos em seu estoque

# briefing
## 1-qual o objetivo real do site 
separar os itens por código, onde cara prateleira terá um código especifico para que seus funcionários saibam osnde se encontram os produtos, para a coleta ou armazenamento

## 2-quem é o publico alvo?
os próprios funcionários da loja

## 3-o que esse site precisa ter?
1. cadastro do funcionário 
2. cadastro dos produtos (ferramentas)
3. localização do código do produto
4. disponibilidade do produto no estoque

# Funcionalidade RF
1. o site deve funcionar apenas em aparelhos especificos somente da loja
2. o site deve o site deve ter uma tabela de cada produto e a sua disponibilidade 
3. o site deve informar se há ou não necesssidade de repor os produtos guardados no estoque

# Usabilidade RNF
1. o site deve ter uma interface intuitiva e simples
2. as principais ações do site devem ser realizadas com no máximo 3 cliques
3. o site deve evitar que o usuário cometa erros, como enviar formulários incompletos

# Confiabilidade RNF
1. O site deve estar disponível 99% do tempo durante o horário de funcionamento
2. O site deve realizar backup dos dados diariamente
3. O site deve evitar que o usuário cometa erros, como enviar formulários incompletos

# Desempenho RNF
1. O site deve suportar até 100 usuários simultâneos sem queda de desempenho
2. O site deve processar requisições sem travamentos
3.o uso de memória não deve ultrapassar limites que prejudiquem o funcionamento

# Suportabilidade RNF
1. O site deve possuir documentação técnica para suporte
2. Erros devem ser identificáveis rapidamente pela equipe de suporte
3. O site deve ser fácil de manter e permitir correções sem afetar outras funcionalidades

# Estrutura (mapa do site)
- página inicial
- formulário de login
- seção de pesquisa do produto dividida em:código, marca e tipo de produto
- catálogo dos produtos
